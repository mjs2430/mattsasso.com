Podcast landing page resources


Use this navigation to get the prebuilt HTML for the 10 markets that have podcasts, the template for the rest of the markets, an example podcast page from Content Studio, and create new cards with the card creators.

Note: the templates for the markets with podcasts have a display error for the top podcast cards, but they display fine once in content studio. It's because they are referencing a stylesheet that is optimized for content studio.

Scenario 1: A market adds a podcast

Summary

Part 1: Market's Top Podcast Section
  1. User needs to create a new “Top Podcast Card” and copy that code
  2. User then navigates to the market's Content Studio Podcasts Free HTML widget
  3. User pastes the code from the Card Creator below the last Top Podcast in the HTML
  4. If the market does not have any previous podcasts, the pasting and HTML editing instructions are as follows:
    1. paste the HTML from the card creator right after the comment <!-- ****** PASTE MARKET PODCAST FROM CARD CREATOR HERE ****** -->
    2. Next, remove the comment tags wrapping the <div> section that has the market's section heading, remove the comment tags so this <div> section goes live, and update the text from "Market Podcasts" to whatever market you are editing, i.e. "Sacramento Bee Podcasts"
    3. Finally, towards the top of the HTML you will see a <style> tag. Just below this you will see a comment /*MARKET ACCENT COLOR */ , change that accent color to the accent color of the market. You can find this info on our Frontify page.
  5. Publish changes
Part 2: Adding a new podcast to the Podcasts Around McClatchy section
  1. User needs to create a new “Around/Archive Podcast Card” and copy that code
  2. On McClatchy DC's FTP server, navigate to mi00s/McClatchyDC/static/creatives/podcasts/around-archive-podcast-iframe.html
  3. edit that file directly, or download it and edit it
  4. Paste the HTML from the card creator tool in the appropriate regional section for that market, i.e. if the Sacramento Bee adds a podcast, add it after the last podcast in the California section
  5. Save that file and re-upload it to the FTP server

Detailed Instructions

Part 1: Market's Top Podcast Section

  1. In the top navigation, click “Top Podcast Card Creator”

  2. From here, you need to enter the new podcast information. The first step is adding the artwork.

    1. Artwork should be 500x500 pixels. If the artwork has an all white background, please edit it so it has some sort of border around it.

    2. Artwork has to be hosted online, so create a new Article in Content Studio that will promote the podcast. Write a short story about it and add the artwork as a related image. Publish this article, and navigate to it in Chrome.

    3. Right click on the artwork and select “Open image in new tab.” Click and hold on the artwork image in that new window, drag your mouse and the image to anywhere on the Podcast Creator page. The image should then display in a new preview card

  3. After the artwork has been added, the next step is adding all the podcast details. You need to enter the image’s Alt text, Podcast title, Podcast description, and URLs to any services that host the podcast audio. This is shown in the image below

  4. Click the "Get Code and Notify McClatchy" button

  5. You should now have a page that looks like this, with a preview of the card and the code output:

  6. Highlight all of the code, right click, and copy it

  7. If the marekt you are working with already has its own podcasts, head over to the market's Content Studio. In the file tree you will see "Podcasts", double or triple click that until it opens a new tab. There you will see Banner Options and one of the nested pages under Banner Options is "Podcasts." Double or triple click that until it opens. Now you will see the HTML for the Podcast landing page. I have left comments in the HTML labeled <-- ****** MARKET PODCAST CARD ****** --> and <-- ****** END MARKET PODCAST CARD ****** --> After the last (there are only 3 at the time of writing this) <-- ****** END MARKET PODCAST CARD ****** --> paste the code you copied from the Card Creator. This is shown in the GIF below:

  8. If the market had no previous podcasts, the instructions are the same up until you get to the point where you have to paste into Content Studio, those instructions are as follows:

    1. In the market's Podcast Free HTML widget page, find the comment < !--********** PASTE MARKET PODCAST FROM CARD CREATOR HERE ************** --> and paste your HTML from the Card Creator tool on the next line

    2. Next, just above the <!--********** PASTE MARKET PODCAST FROM CARD CREATOR HERE ************** --> comment, you will see a commented out section labeled <!--**** Update this with Market name ****-->

      1. Remove the beginning of the comment tag <!-- that is just before the <div> tag that is under the <!--**** Update this with Market name ****-->

      2. Remove the end of the comment tag --> that is just after the ending </div> tag at the end of this "Market Podcasts" section. What these comment tags look like is shown below

      3. Change the text "Market Podcasts" to whatever market you are working on, i.e. "Sacramento Bee Podcasts"

    3. Next, in line 4, you will see the start of a "style" tag, and in that style section you will see the comment /* MARKET ACCENT COLOR */

    4. That 6 digit HEX code needs to be changed to the HEX code associated with the specific market's accent color

    5. To find this HEX color code, head to our McClatchy Frontify Style Guide

    6. In the left navigation, you will see a list of all our markets, click on the market you are editing, now you will see two colors, one primary and one accent

    7. Copy that HEX code from the "#" all the way to the last digit. Paste it over the HEX code in the style section of our HTML in Content Studio

  9. Click "Publish" and you are done with part 1!

Part 2: Adding a new podcast to the Podcasts Around McClatchy section (Done by the Design Team for now)

  1. In the top navigation, click "Around/Archive Podcast Card Creator"

  2. Add all the podcast details just like in Part 1, but you do not need an image for artwork

  3. Highlight and copy all of the output code. The next step is where Part 2 varies from Part 1

  4. This process can vary depending on the FTP client you use, but for this document demo below, I am using Filezilla. The first step is to log in with McClatchy DC's FTP credentials

    1. On the right panel of Filezilla, navigate to mi00s/McClatchyDC/static/creatives/podcasts. Here you will see a file named "around-archive-podcast-iframe.html"

    2. Double click that file and it will download it to whatever folder on your computer that you have open in the left panel

    3. Now, using Finder (Mac) or Explorer (Windows), navigate to where you downloaded the file. In my example below, I downloaded it to my Documents folder

    4. Right click it, and open it with your favorite text editor. I used Adobe Brackets, but I also recommend Notepad++ or Sublime Text

    5. In this HTML you will see comments that tell you what region the code block is for, such as <!--- Midwest ---->

    6. Depending on what region your market is in, i.e. California, The Carolinas, The Midwest, The Northeast, or the Southeast, you will change where you need to paste the output HTML from the Card Creator Tool

    7. You will need to go to the area in the HTML that is specific to your market. Each podcast is contained in a <li> </li> tag. After the last podcast, after its ending </li> tag, paste your HTML code from the Card Creator Tool.

    8. We will be using the Carolinas as the region in the example below:

    9. Next, back in the top left area of the Filezilla toolbar you will see a blue and a green arrow creating a sort of circle, this is the Refresh button. Click that to make sure it has the most recent save of the HTML file you just edited/saved

    10. Now that it's refreshed, find your "around-archive-podcast-iframe.html" file in the left pane. Double click it. This will send it over Filezilla to the FTP server

    11. Filezilla will warn you that this file already exists, the "overwrite" radio button should already be selected, so just click "Ok" to upload the file and overwrite the old one. Now you're done!

Scenario 3: A podcast gets archived

Summary

Part 1: Removing the podcast from its market page
  1. User needs to navigate to the podcast free HTML page in the market's Content Studio
  2. User then identifies the <div> tag that the card is in, and delete all of it
  3. Publish changes
Part 2: Moving the podcast to the Archive section
  1. If it is a McClatchy DC podcast:
    1. On McClatchy DC's FTP server, navigate to mi00s/McClatchyDC/static/creatives/podcasts/mcdc-podcast-iframe.html
    2. Identify the <div> tag that the podcast card is in, and delete it
    3. Use the Around/Archive Card Creator Tool to create the HTML for the podcast
    4. On McClatchy DC's FTP server, navigate to mi00s/McClatchyDC/static/creatives/podcasts/around-archive-podcast-iframe.html
    5. Towards the bottom of the HTML, you will see a section titled <!---- ******* Archived Podcast Section ******* ---->
    6. Find the section for McClatchy DC
    7. Paste the <li> tag you copied from the card creator after the last </li>
    8. Save and re-upload the file to the FTP server
  2. If it is a podcast from any other market:
    1. On McClatchy DC's FTP server, navigate to mi00s/McClatchyDC/static/creatives/podcasts/around-archive-podcast-iframe.html
    2. In the <!---- ***** ********* Around McClatchy Podcast Section ********* *******----> find the region of the market your podcast is in
    3. Here you will see one or more <li> tags that contain each podcast. Find the podcast that you are working with
    4. Highlight the entire <li> to </li> and cut it
    5. Towards the bottom of the HTML, you will see a section titled <!---- ******* Archived Podcast Section ******* ---->
    6. Find the section for region you are working with
    7. Paste the <li> tag you cut from above after the last </li>
    8. Save and re-upload the file to the FTP server
